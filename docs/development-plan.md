# VOXO 개발 계획서

작성일: 2026-03-10

## 1. 목표

VOXO를 "화면이 있는 시제품" 수준에서 "실제로 운영 가능한 음악 매거진 CMS" 수준으로 끌어올리는 것이 목표입니다.

이번 계획서는 아래 3가지를 중심으로 작성합니다.

- 무엇을 먼저 고칠지
- 어떤 기능을 다음으로 붙일지
- 어떤 순서로 진행해야 위험이 적은지

## 2. 개발 원칙

- 먼저 안정화, 그다음 기능 확장
- 관리자 운영 흐름을 우선 완성
- AI 기능은 화려함보다 실패율 감소를 우선
- 문구/인코딩/타입 오류를 초기에 정리
- 배포 가능 상태를 계속 유지

## 3. 단계별 계획

### 1단계: 안정화

목표:
기존 기능이 빌드와 린트 기준으로 안정적으로 유지되도록 정리합니다.

작업:

- 린트 오류 제거
- 빌드 오류 제거
- 관리자 화면 깨진 문구 정리
- 공통 타입 정리
- AI 초안 생성 로직 공통화
- 문서 정리

완료 기준:

- `npm run build` 통과
- `npm run lint` 오류 0건
- 관리자 핵심 화면이 최소한 읽을 수 있는 상태

현재 상태:
대부분 완료. 남은 항목은 이미지 최적화 경고와 일부 관리자 문구 정리입니다.

### 2단계: 운영 기능 완성

목표:
관리자 화면만으로 실제 운영이 가능하게 만듭니다.

작업:

- 뉴스레터 실발송 서비스 연동
- 설정 페이지 실사용 항목 정리
- 설정값이 실제 기능에 반영되도록 연결
- 구독자 상태 관리 보강
- 발송 성공/실패 처리 보강

완료 기준:

- 테스트 뉴스레터 발송 가능
- Gemini API Key, 글로벌 Spotify URL 등 주요 설정 반영 확인
- 관리자 기준 운영 루틴 가능

### 3단계: 콘텐츠 제작 고도화

목표:
콘텐츠 작성 속도와 품질을 함께 높입니다.

작업:

- 에디터 UX 개선
- 미리보기 강화
- SEO 메타데이터 강화
- 관련 글 추천
- 태그/카테고리 연결 완성도 개선

완료 기준:

- 글 작성부터 발행까지 흐름이 자연스러움
- 포스트별 메타데이터 일관성 확보

### 4단계: 공개 사이트 UX 고도화

목표:
방문자가 보는 화면의 완성도를 높입니다.

작업:

- 모바일 메뉴 고도화
- 검색 UX 개선
- 공유 기능 실연결
- 빈 화면/오류 화면 정리
- 이미지 최적화

완료 기준:

- 모바일과 데스크톱 모두 자연스럽게 사용 가능
- 공개 화면의 체감 완성도 상승

### 5단계: AI/분석 고도화

목표:
VOXO만의 생산성 기능을 강화합니다.

작업:

- AI 생성 이력 관리
- 재시도/실패 처리 개선
- Spotify 데이터 캐시 보강
- 관리자 분석 카드 고도화
- 운영 로그 정리

완료 기준:

- AI 초안 생성 실패율 감소
- 운영자가 데이터 흐름을 더 쉽게 파악 가능

## 4. 다음 실제 작업 우선순위

가장 추천하는 다음 순서는 아래와 같습니다.

1. 관리자 설정 페이지 문구/인코딩 정리
2. 관리자 공통 번역 문자열 정리
3. 뉴스레터 실발송 연동
4. 이미지 최적화 적용
5. 공유 기능 및 SEO 보강

## 5. 비고

현재 기준으로 가장 큰 위험은 "기능 부족"보다 "운영 준비 미완성"입니다.

그래서 앞으로의 개발은 새 페이지를 많이 추가하는 것보다, 이미 있는 기능을 실제 운영 가능한 수준으로 마감하는 방식이 가장 효율적입니다.
