# VOXO 실행 로드맵

## 1. 작업 순서
이 프로젝트는 아래 순서로 진행하는 것을 기준으로 합니다.

1. 구조 최적화
2. 글쓰기/편집기 개선
3. AI 기사 생성 보강
4. 관리자 기능 보강
5. 공개 페이지 미완성 기능 보강
6. 운영 안정화 및 최종 점검

## 2. 단계별 계획

### Phase 1. 구조 최적화
목표:
- 큰 페이지를 기능 단위로 분리
- 액션 파일 책임 분리
- 공통 UI/검증/문구 구조 정리

완료 기준:
- 관리자 핵심 페이지가 더 작은 컴포넌트들로 나뉨
- 설정/뉴스레터/에디터/AI 영역의 공통 패턴이 정리됨

### Phase 2. 편집기 개선
목표:
- 글 작성 흐름을 직관적으로 개선
- 이미지, 태그, 음악 설정을 쉽게 만들기

세부 작업:
- 본문 이미지 삽입
- 링크 삽입 개선
- 태그 선택 UX 개선
- Spotify 연결 UX 개선
- 자동 임시저장
- 미리보기 패널

완료 기준:
- 운영자가 실제로 글 쓰는 시간이 줄어듦
- 작성 중 실수나 누락이 줄어듦

### Phase 3. AI 기사 생성 보강
목표:
- AI가 만든 초안을 사람이 빠르게 손볼 수 있게 만들기

세부 작업:
- 제목/요약/태그 자동 제안
- 음악 메타 자동 연결
- 이미지 후처리 흐름 추가
- 에디터 연동 강화

완료 기준:
- AI 결과물을 바로 편집기로 넘겨 수정/발행 가능

### Phase 4. 관리자 보강
목표:
- 운영 화면을 더 명확하고 안전하게 만들기

세부 작업:
- 설정 변경 로그 고도화
- 실패 경고 강화
- 뉴스레터 운영 기능 추가
- 보안/점검 모드 UX 보강

### Phase 5. 공개 페이지 보강
목표:
- 실제 방문자 경험 개선

세부 작업:
- 관련 글/추천 글 개선
- 태그 탐색 강화
- 검색 개선
- 모바일 UX 보정
- SEO 강화

### Phase 6. 안정화
목표:
- 배포 전 최종 안정화

세부 작업:
- 전체 회귀 점검
- 주요 페이지 QA
- 관리자 핵심 흐름 테스트
- 문구/인코딩 정리

## 4. 권장 진행 방식
- 한 번에 큰 개편을 하지 말고, Phase 단위로 완료 후 다음 단계로 이동합니다.
- 특히 Phase 1과 Phase 2는 바로 이어서 진행하는 것이 좋습니다.
- 기능 추가보다 먼저 "수정하기 쉬운 구조"를 만드는 것이 핵심입니다.

## 5. 다음 실제 작업 시작점
바로 다음 작업은 아래 순서를 권장합니다.

1. `admin/settings`, `admin/editor`, `admin/ai-desk` 분리 설계
2. 에디터에서 이미지/링크/태그/Spotify UX 개선
3. AI 생성 결과를 에디터 후처리 흐름과 더 강하게 연결

이 문서를 기준으로 이후 실제 개발을 단계별로 따라가면 됩니다.
