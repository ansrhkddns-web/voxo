# VOXO 기능 보강 계획서

## 1. 목표
코드 구조를 먼저 정리한 뒤, 아래 4개 영역을 중심으로 기능을 실사용 가능한 수준까지 끌어올립니다.

1. 공개 페이지
2. 관리자 페이지
3. 글쓰기/편집기
4. AI 기사 생성 및 후처리

## 2. 공개 페이지 보강 항목

### 2-1. 메인/카테고리/상세 페이지
- 추천 글 노출 기준 고도화
- 최신 글/인기 글 분리
- 태그/카테고리 이동 UX 개선
- 공유 기능 보강
- 검색 정확도 개선

### 2-2. 미비 기능
- 관련 글 추천 정교화
- 태그 기반 탐색 강화
- 모바일 사용성 개선
- SEO 메타데이터 보강
- 외부 공유용 썸네일/설명 일관화

## 3. 관리자 페이지 보강 항목

### 3-1. 대시보드
- 최근 작업 내역 더 명확히 표시
- 발송 실패 원인 분류
- 점검 모드/설정 상태 경고 강화
- 콘텐츠 상태 요약 카드 고도화

### 3-2. 설정
- 변경 로그 화면 별도 조회
- 민감 설정 저장 전 검증 강화
- 환경변수 누락 경고 표시

### 3-3. 뉴스레터
- 템플릿 저장
- 테스트 발송 이력 분리
- 예약 발송 준비 구조
- 실패 재시도 흐름

## 4. 글쓰기/편집기 보강 항목

### 4-1. 가장 시급한 문제
- 글 작성 흐름이 불편함
- 한 화면에 입력 요소가 너무 많음
- 이미지 첨부와 본문 편집 흐름이 분리되지 않음
- 태그/음악/메타 설정이 느리고 직관적이지 않음

### 4-2. 꼭 추가해야 할 기능
- 표지 이미지 업로드 UX 개선
- 본문 중간 이미지 삽입
- 하이퍼링크 삽입/편집 UX 개선
- 태그 추천 및 빠른 선택
- Spotify URL/URI 자동 판별
- 미리보기 패널
- 자동 임시저장
- 발행 전 체크리스트

### 4-3. 편집기 개선 방향
- 기본 정보 단계
- 본문 작성 단계
- 미디어/태그 단계
- 발행 검토 단계

즉, 지금의 단일 대형 편집 화면을 "작업 흐름형 편집기"로 바꾸는 것이 목표입니다.

## 5. AI 기사 생성 기능 보강 항목

### 5-1. 현재 부족한 부분
- 기사 생성 후 이미지 연결 흐름 약함
- 태그/음악/링크 메타가 자동으로 충분히 채워지지 않음
- 생성 결과를 에디터에서 후편집하기 불편함

### 5-2. 추가 목표
- AI 기사 생성 시 추천 제목/요약/태그 자동 제안
- Spotify 링크 자동 추출 또는 추천
- 표지 이미지 후보 연결
- 카테고리 추천
- SEO 태그 초안 자동 생성
- 생성 완료 후 곧바로 에디터 편집 화면으로 이동

### 5-3. 이미지 관련 보강
- 기사 생성 후 이미지 첨부 슬롯 제공
- 커버 이미지와 본문 이미지 구분
- 추후 외부 이미지 API 또는 수동 업로드 연계 가능 구조 준비

## 6. 우선순위

### 1순위
- 편집기 사용성 개선
- AI 생성 결과 후처리 개선
- 관리자 구조 리팩터링

### 2순위
- 공개 페이지 미완성 기능 보강
- 뉴스레터 운영 기능 강화

### 3순위
- 예약 발송
- 고급 추천 로직
- 통계/리포트 고도화

## 7. 완료 기준
- 글 작성 시간이 지금보다 확실히 줄어들 것
- 관리자 화면에서 운영 상태를 바로 이해할 수 있을 것
- AI 생성 결과를 "바로 발행 가능한 초안" 수준으로 끌어올릴 것
- 공개 페이지 탐색 흐름이 지금보다 자연스러워질 것
