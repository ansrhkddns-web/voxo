# AI Music Curator System

## 목적
- VOXO의 AI 글쓰기를 단순 요약기가 아니라 감성적이고 공감 가능한 뮤직 큐레이터로 고도화한다.
- 글 생성 전에 `큐레이팅 프로필`과 `기사 분량`을 선택하게 해서, 같은 곡이라도 목적에 맞는 전혀 다른 결과물을 만든다.
- 이미지 선택, Spotify 매칭, 태그 매칭까지 포함한 한 번에 완성되는 작성 흐름을 만든다.

## 현재 적용된 구조
- 글 생성 전 선택 항목
  - `Curation Profile`
  - `Article Volume`
  - `Category`
  - `Tone`
  - `Image Style`
  - `Link Priority`
  - `Draft Direction Note`
- AI가 실제로 따르는 핵심 작성 규칙
  - 곡의 감정과 사운드 디테일을 근거로 서술
  - 장문 피처 기사 길이 기준으로 작성
  - 프로필별 `권장 표현`, `금지 표현`, `스타일 예시`를 참고해 문체를 안정화
  - 카테고리별 기사 역할에 따라 논조와 구조를 다르게 조정
  - 초안 작성 뒤 `2차 리라이트` 단계에서 제목, Hero Excerpt, 문단 흐름을 다시 정리
  - 본문 안에 이미지 플레이스홀더, 중간 유튜브 원문 링크, 외부 URL 남기지 않음
  - 실제 태그 라이브러리에서 장르 태그만 선별

## 카테고리별 논조 규칙
- `Reviews`
  - 비평과 감정, 근거를 함께 밀어붙이는 정통 리뷰 논조
- `News`
  - 팩트 전달이 먼저고, 그다음 의미를 해석하는 뉴스형 논조
- `Editor's Pick`
  - 추천과 설득 중심의 에디터 추천 논조
- `Features`
  - 감성, 분석, 맥락을 균형 있게 담는 피처형 논조
- `Focus`
  - 한 가지 핵심 포인트를 집중적으로 밀어붙이는 논조
- `Archives`
  - 재발견과 기록의 의미를 살리는 아카이브 논조
- `Cover Story`
  - 메인 특집처럼 넓고 입체적으로 전개하는 논조

## 카테고리별 문단 설계 규칙
- 카테고리마다 `첫 문단 시작 방식`이 다르다.
  - `News`: 핵심 사실부터 바로 시작
  - `Reviews`: 감정과 긴장부터 시작
  - `Editor's Pick`: 추천 이유를 바로 제시
  - `Archives`: 재발견과 기억의 감각으로 시작
- 카테고리마다 `마지막 문단 마무리 방식`도 다르다.
  - `News`: 이 변화가 왜 중요한지 짧게 정리
  - `Reviews`: 분석을 바탕으로 비평적 결론 제시
  - `Editor's Pick`: 플레이 버튼을 누르게 하는 여운
  - `Cover Story`: 더 큰 문화적 의미까지 확장

## 카테고리별 제목 / Hero Excerpt 규칙
- 카테고리마다 `제목`의 성격이 다르다.
  - `News`: 핵심 사실을 앞세운 정보형 제목
  - `Reviews`: 감정과 평가 각도를 담는 비평형 제목
  - `Editor's Pick`: 추천 욕구를 자극하는 권유형 제목
  - `Focus`: 무엇을 파고드는지 바로 보이는 각도형 제목
- 카테고리마다 `Hero Excerpt`도 다르다.
  - `News`: 사실 요약 + 왜 중요한지 한 줄
  - `Reviews`: 감정과 판단을 함께 암시
  - `Editor's Pick`: 왜 지금 들어야 하는지 바로 전달
  - `Archives`: 다시 꺼내 들어야 하는 이유를 회고적으로 전달

## 카테고리별 본문 전개 규칙
- 카테고리마다 `중간 문단의 흐름`도 다르게 설계한다.
- 예시
  - `Reviews`: 감정 인상 -> 클로즈 리스닝 -> 비평 판단
  - `News`: 핵심 사실 -> 배경 설명 -> 의미 정리
  - `Editor's Pick`: 추천 포인트 -> 어떤 청자에게 맞는지 -> 왜 기억에 남는지
  - `Archives`: 재발견 감각 -> 당시 맥락 -> 지금 다시 남는 이유
  - `Features`: 서사 -> 분석 -> 맥락 -> 정리

## 이미지 큐레이션 규칙
- 대표 이미지 후보: 실존 아티스트 이미지 5개 검색
- 본문 이미지: 항상 2개만 사용
- 본문 이미지 1장을 고른 뒤 다시 검색하면
  - 이미 고른 이미지는 후보에서 제외
  - 새로운 후보만 다시 검색
- 본문 이미지 2장을 모두 고르면
  - 기사 본문 고정 위치에 자동 배치

## 권장 기본값
- 기본 프로필: `Deep Review`
- 기본 분량: `Feature`
- 이 조합이 현재 VOXO 기준으로 가장 완성도 높은 기본 조합이다.

## 운영자가 확인할 포인트
- `Deep Review`
  - 곡 자체를 깊이 파고드는가
  - 감정 표현이 구체적 사운드 근거와 연결되는가
- `Editor Pick`
  - 추천 이유가 빠르게 전달되는가
  - 낯선 곡도 들어보고 싶게 만드는가
- `Artist Focus`
  - 곡을 통해 가수 전체를 읽게 만드는가
- `Mood Essay`
  - 감성적이지만 공허하지 않고 실제 음악 디테일을 붙잡고 있는가

## 새로 추가된 문체 제어 장치
- `Preferred Direction`
  - 프로필에 맞는 표현 방향을 AI에게 힌트로 준다.
- `Avoid Tone`
  - 흔하고 밋밋한 표현, 홍보문 같은 표현, 팬 위키 같은 문체를 피하게 만든다.
- `Style Example`
  - 그대로 베끼는 문장이 아니라, 문장의 온도와 결을 맞추는 기준으로 사용한다.

## 2차 리라이트 단계
- 초안 작성 직후 한 번 더 다듬는다.
- 이 단계에서 정리하는 것
  - 제목 규칙
  - Hero Excerpt 규칙
  - 첫 문단 시작 방식
  - 중간 문단 전개 흐름
  - 마지막 문단 마무리 방식
- 목적
  - 초안은 살아 있게 두고, 완성도는 편집 기사처럼 끌어올리는 것

## 다음 고도화 권장 순서
1. 프로필별 기사 예시 3~5개를 모아 학습용 레퍼런스 세트 구축
2. 프로필별 금지 표현, 권장 표현 사전 추가
3. 에디터에서 프로필 변경 시 추천 문체/후킹 문구 자동 재생성
4. 발행 전 품질 체크에서 프로필별 체크리스트 분기
